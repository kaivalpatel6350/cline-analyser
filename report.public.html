Cognitive skills

Add a validation gate before new directives

You: Self-correction rate jumped to 10.5% (recent) from 4.45% (prior) — a +6.05 delta.

Why: Rising self-correction suggests you’re issuing directives on partially verified state, then revising. That compounds in long sessions.

Try: For 2 weeks, before any new change request, ask 1 verification question or request a brief state summary. Target: validation in ≥1 of every 5 substantive turns. Watch if self-correction rate drops from 10.5% toward prior levels.

Let tool work breathe (reduce depth‑1 interrupts)

You: tool_depth_at_followup p50 = 1, p90 = 1.

Why: You often reply after a single tool call — likely cutting off retrieval/analysis mid‑stream, increasing rework.

Try: In file/tool tasks, wait until ≥2 tool calls complete (or “analysis complete”) before replying. Track 20 sessions; goal: raise your median follow‑up depth from 1 → ≥2.

Manage cognitive load in long hauls

You: 2,882.45 total active hours over 605 sessions (~4.8h/session). p90 session length ~6h.

Why: Long sessions increase decision fatigue → more corrections, shorter messages, reactive loops.

Try: For 14 days, cap at 3 sessions >2h/day. Log daily deep sessions. Aim to reduce fatigue without lowering weekly output.

Prompt craft

Make execution requests more decisive

You: Recent messages are shorter (182.99 vs 240.11 words, −57.12).

Why: Shorter prompts can be sharper — or under‑specified. Pair brevity with explicit outcomes to avoid back‑and‑forth.

Try: Start new threads with a verb + outcome + constraint block:

Refactor X to Y.
Constraints: A, B, C.
Success = tests pass + latency < N ms.

Measure: average follow‑ups per task over 2 weeks.

Ground debug work in artifacts

You: Heavy fix/edit pattern: 5,956 replace_in_file + 4,821 read_file calls; 19,248 total tool calls.

Why: Your workflow is artifact-heavy. Ambiguity costs more here than in ideation.

Try: In bug reports, always include exact error text or a minimal code block. Track avg follow‑ups per bug for 2 weeks; target −10%.

Add a 30‑second send check

You: Typos increased to 13.73 per 100 msgs (recent) from 12.05 (prior), +1.68.

Why: In constraint-heavy debug threads, small wording errors create misalignment.

Try: Before sending, re-read once for constraints/negations. Goal: bring typos back below 12 per 100 msgs over the next window.
